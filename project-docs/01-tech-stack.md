# Tech Stack — What Each Technology Is Used For

Homies has **two front-end clients** (Flutter mobile + React web) sharing one **Firebase** backend. There is **no custom backend server, no self-hosted database, and no REST/GraphQL API of our own** — clients talk directly to Firebase.

---

## Summary table

| Layer | Technology | Used for |
|-------|-----------|----------|
| **Mobile app** | Flutter (Dart), SDK ^3.10.7 | The primary product — full cross-platform app (iOS/Android) |
| **Web app** | React 19.2.6 + Vite 8.0.12 | Browser client / marketing landing + app screens |
| **Backend** | Firebase (BaaS) | Replaces a custom server — auth, data, file storage |
| **Auth** | Firebase Authentication | Email/password sign-up & login |
| **Database** | Cloud Firestore (NoSQL) | User profiles & cloud-synced documents (`users/{uid}`) |
| **File storage** | Firebase Storage | Receipts, ID docs, bond proof, task photos, issue photos |
| **Local persistence** | `shared_preferences` (mobile) / `localStorage` (web) | Offline-first app state cache |

| **Web routing** | React Router DOM 7.15.0 | Client-side page navigation |
| **Mobile state** | `ChangeNotifier` + `InheritedNotifier` | App-wide state store (`HomiesState` / `HomiesScope`) |
| **Web state** | React Context API | App-wide state store (`HomiesContext`) |
| **Build (mobile)** | Flutter/Dart toolchain | Compile & bundle the mobile app |
| **Build (web)** | Vite + `@vitejs/plugin-react` 6.0.1 | Dev server & production bundling |
| **Linting (mobile)** | `flutter_lints` 6.0.0 | Dart code quality |

## Frontend — Web

- **React** `19.2.6` — UI component framework.
- **React Router DOM** `7.15.0` — client-side routing between pages.
- **Vite** `8.0.12` — build tool and dev server; `@vitejs/plugin-react` `6.0.1` for React + Oxc.
- **@types/react** `19.2.14`, **@types/react-dom** `19.2.3` — TS type definitions (dev only).
- **ESLint** `10.3.0` with `@eslint/js`, `eslint-plugin-react-hooks`, `eslint-plugin-react-refresh` — linting.
- Config: `vite.config.js`, `eslint.config.js`, entry `index.html`.

> Note: the current web build serves a marketing **landing page**; the full React app screens (Dashboard, Bills, etc.) exist in `frontend_web/src/pages/` and mirror the mobile app, but are not all routed in the live build.

## Frontend — Mobile (primary product)

- **Flutter** (SDK `^3.10.7`) / **Dart** — cross-platform app framework.
- **Material Design 3** — built-in theming (`lib/theme.dart`).
- **go_router** `14.6.2` — routing and navigation.
- **shared_preferences** `2.3.3` — local persistence (key `homies-mobile-v2`).
- **intl** `0.20.1` — date/currency formatting (locale `en_AU`).
- **file_picker** `8.1.6` — selecting files/photos from the device (receipts, ID docs).
- **uuid** `4.5.1` — generating unique IDs for records.
- **cupertino_icons** `1.0.8` — iOS-style icons.
- Dev: **flutter_lints** `6.0.0`, **flutter_test**.

## Backend / API

- **No custom backend server. No custom REST or GraphQL API.**
- **Firebase** is used as a complete Backend-as-a-Service; clients call Firebase SDKs directly.
- **Firebase Authentication** — email/password login (web `firebase` `12.13.0`; mobile `firebase_auth` `6.5.1`).
- **Cloud Firestore** — the "API" surface for cloud data: reads/writes user profile documents and auth state under `users/{uid}` with server timestamps.
- Mobile Firebase packages: `firebase_core` `4.9.0`, `firebase_auth` `6.5.1`, `cloud_firestore` `6.4.1`, `firebase_storage` `13.4.1`.
- Config: `frontend_app/lib/firebase_options.dart` (generated by FlutterFire CLI).

## Database

- **Cloud Firestore** (NoSQL document DB) — cloud-stored user profiles & auth state.
- **Firebase Storage** — binary file storage (receipts, ID documents, bond/advance-rent proofs, cleaning photos, issue photos).
- **Local-first cache:** most app state (bills, chores, listings, messages, etc.) is held in the client store and persisted locally — mobile via `shared_preferences` (`homies-mobile-v2`), web via `localStorage` (`homies-state-v3`). Demo/seed data lives in `lib/state/seed.dart` (mobile) and `frontend_web/src/data/mockData.js` (web).

## State Management

- **Mobile:** single source of truth `HomiesState` (extends `ChangeNotifier`), exposed to the widget tree via `HomiesScope` (`InheritedNotifier<HomiesState>`). Mutations go through `state.mutate(() => …)`, which notifies listeners and re-persists. No external state package.
- **Web:** `HomiesContext` provider via React Context API + `useContext`, persisted to `localStorage`.

## Build / Tooling

- **Web:** Vite (`npm run dev` / `npm run build`), npm + `package-lock.json`, ESLint.
- **Mobile:** Flutter/Dart toolchain (`flutter run` / `flutter build`), `pubspec.yaml` + `pubspec.lock`, FlutterFire CLI for Firebase config.
